1. Getting started

On first boot you'll be redirected to /setup to create the admin login and (optionally) paste an AcoustID key. After login, head to Library → Manage libraries and add at least one path on disk that dragontag should manage. This becomes the destination root for everything you tag.

Three ways to feed files into the pipeline:

  • Upload — drag files into the Dashboard upload card.
  • Drop folder — copy files into the path mounted at /drop; the watcher catches them.
  • Full library re-tag — point a server-side path at the bulk-retag form to enqueue every audio file inside.

2. Tagging workflow

Drop file  →  Read existing tags + filename clues
           →  MusicBrainz text search
           →  [empty] AcoustID fingerprint fallback
           →  Score candidates  →  Pick best
           →  [score ≥ threshold] Fetch full MB data
                                  Fetch cover art (Cover Art Archive)
                                  Fetch lyrics (LRCLIB) + detect advisory
                                  Apply smart formatting (optional)
                                  Write tags  →  Move to library
           →  [score < threshold or missing data] Review queue

High-confidence matches flow through without input. Anything else lands in Review where you confirm or override.

3. Dashboard

  • Stats bar — total tracks, albums, artists currently indexed.
  • Tag audio files — upload individual files or kick off a folder re-tag.
  • Library stats — top artists, explicit-count, lyrics-coverage, average track length.
  • Recent jobs — last five jobs with status, click any row for full detail.

4. Keyboard shortcuts

Universal keys work on every page; press ? any time to bring up this list in-app. Each page's status bar (bottom of the screen) shows the keys specific to that page.

  • / — focus that page's search/filter box.
  • ? — toggle the key-reference overlay.
  • g then a letter — jump to a page: d dashboard, l library, q queue, c changes, h schedule, s settings, o docs.
  • Library — hover a row, then space toggles its checkbox, Enter opens its edit modal, r submits the re-tag form.
  • Queue — hover a row, then space toggles its checkbox, a applies it, x cancels it.
  • Changes — hover a row, then v reverts it, b moves it back to its original directory.
  • Job detail — r requeues the job, Esc returns to the dashboard.
  • Settings — hover a checkbox label, then space toggles it; Alt+s saves the form.

5. Settings

Each field has a short description directly underneath it. Highlights:

  • Identification — AcoustID toggle, auto-apply threshold, watcher on/off, global dry-run. The dry-run checkboxes on Library actions are per-run only and never change this global default.
  • Smart formatting — Title Case, qualifier wrapping, grammar correction (see §12).
  • Filename templates — single-disc, multi-disc filename, and multi-disc folder patterns. Use the token chips to insert placeholders; a live preview shows the result.
  • Genre options — cap and casing.
  • Multi-value separators — joiner per tag (default // for ARTIST, ; elsewhere).
  • Skip fields — checkbox grid: any checked field is omitted at write time across every format.
  • Webhooks — Discord-compatible URL with on-done / on-error toggles and a Test button.
  • Maintenance — how many Changes-page entries to keep (0 = unlimited) and a 0–4 log verbosity slider (silent / errors / warnings / info / debug) applied immediately on save.
  • Scan filters — three lists that exclude files/directories from all scan, ingest, and bulk-retag operations. A Clear all scan filters button (with confirmation) empties them in one go.
    • File filter patterns — one regex per line, matched against filenames. E.g. \.ini$ skips all .ini files; Thumbs\.db$ skips Thumbs.db.
    • Excluded directories — one absolute path per line. Files inside these directories are never scanned or ingested.
    • Excluded files — one absolute file path per line. Files returned to their original directory via "Move back" are added here automatically so they aren't immediately re-ingested; you can also edit the list by hand.
  • Backup & restore — see §11.
  • Time zone — controls how timestamps (changes, jobs, dashboard) are displayed. Locked to the container's TZ environment variable when set (always wins); otherwise editable here, defaulting to UTC.

6. Library

  • Scan library — refresh the DB from on-disk tags. Run after manually editing tags outside dragontag.
  • Organize library (batch) — structural cleanup chain: move files to match templates, fix disc folders, normalize filenames, extract embedded covers, prune junk & empty folders, find duplicates, find missing tracks. Files are moved/renamed and junk + empty directories deleted; audio files are never deleted.
  • Full library re-tag (batch) — validate tags, re-tag advisories, fix genres, recompute ReplayGain, then queue every file through the full identify → tag → move pipeline. Supports a per-run dry-run override.
  • Nuclear option — the Organize batch followed by the Re-tag batch, as one job, with a Fix album splits pass right after the re-tag pipeline so albums whose tracks matched different MusicBrainz editions come out unified. The most invasive thing dragontag can do.
  • Individual library actions — each runs on its own via run, or tick several and Queue selected to run them sequentially as a single chained job:
    • Fix disc folders — normalize Disc N subfolders to the configured template; flatten stray single-disc folders.
    • Normalize filenames — lowercase extensions, strip trailing dots/spaces, collapse double spaces. No re-tagging.
    • Extract embedded covers — write a cover.jpg from each album's embedded picture when none exists on disk.
    • Fetch cover art — Cover Art Archive fetch for tracks with an MB album ID.
    • Fetch lyrics — LRCLIB fetch + embed, no identify.
    • Tag advisories — re-run the explicit-content classifier on already-embedded lyrics.
    • Fix genres — backfill missing genres from MusicBrainz community tags (the recording, then the release-group as a fallback) for tracks that have none. Only fills empty genres, never overwrites an existing one; tracks without a MusicBrainz id are left alone. Uses the network.
    • Recompute ReplayGain — invokes rsgain or loudgain per album folder if installed.
    • Verify file integrity — opens every file via mutagen, logs unreadable files.
    • Validate tags — report missing core tags, mis-encoded text and impossible track/disc numbers (report only).
    • Find duplicates — report likely duplicates by MB recording ID or matching artist/title/duration (report only).
    • Prune junk & empty folders — delete OS litter (Thumbs.db, .DS_Store, *.tmp …) and completely empty directories. Also reports (never deletes) dead folders — no audio anywhere below them but leftover cover.jpg/.lrc files — and album folders with only Disc NN subfolders where disc 1 is missing.
    • Find missing tracks — compare local counts to the MB release totals; incomplete albums land on the Library page's Incomplete tab with the missing track titles.
    • Fix album splits — repair albums whose tracks were identified against different MusicBrainz editions of the same release group (multiple album IDs / titles / track totals / covers, shown as several albums by players). Elects the edition covering the most of your tracks (then Official status, then size), fully re-tags every track against it from MusicBrainz — preserving embedded lyrics/advisories and keeping existing art when no canonical cover is available — and merges the files into one folder. Edition-exclusive bonus tracks and protected tracks are left untouched; albums without MusicBrainz IDs get the offline majority vote below. Uses the network.
    • Fix album/folder consistency — offline: tracks sharing a release group (or a normalized album+artist match — now folding case, curly quotes, Unicode dashes, and iTunes-style - Single/- EP suffixes) with disagreeing album/album-artist tags are patched to the majority value and outlier files moved into the single canonical folder. Only album/album-artist are touched.
    • Fix artist folders — offline: collapse duplicate artist folders that differ only by capitalization (fakemink/Fakemink), punctuation/Unicode (curly quotes, dashes, ® marks) or MusicBrainz alias (FERG/A$AP Ferg) into one canonical folder per artist. Elects the majority album-artist spelling among your files (stylized-lowercase names are kept — no forced capitals), rewrites outlier album-artist tags and moves each file under the canonical folder, leaving albums untouched, then renames case-only folder variants and prunes emptied folders. Protected tracks are skipped. Runs before Fix album/folder consistency in the batches, so artist folders are unified before albums are recomputed under them.
  • Incomplete tab — albums whose local track count is below the MusicBrainz total, with per-row MB links and dismiss. Repopulated on every "Find missing tracks" run.
  • Track table — click any column header to sort. Use the per-row checkboxes plus "Re-tag selected" to enqueue specific tracks. Choose your page size (10 / 25 / 50 / 100 / 200). The × at the end of a row removes a stuck entry from the DB only — the file is never touched, and a re-scan re-adds it.
  • Edit a track — opens a modal with every field editable by hand. The header has protect (exempts the track from library batch actions overwriting your edit) and fetch lyrics (LRCLIB, this track only) buttons. A link to album picker lets you attach the track to an album already in your library — it copies that album's album, album artist, disc/track totals, and MusicBrainz album IDs onto the track with no new MusicBrainz lookup; title, artist, and track number are left as you have them.

7. Queue — review section

Review and Jobs now live together on the Queue page (the old /review and /jobs URLs redirect there). A file lands in the review section when it scored below the threshold, no candidate was found, the destination already exists, RELEASETYPE was missing, or dry-run mode was on. From there you can:

  • Search MusicBrainz manually by title / artist / album or a pasted MB URL / ID — click Search (or press Enter in any field) to run the query.
  • Pick any of the top candidates and apply.
  • Override RELEASETYPE, upload custom cover art, or pick a different cover-art thumbnail.
  • Resolve conflicts with Replace, Rename, or Skip.
  • Bulk-select rows and click Apply top candidate to clear them in one go.

8. Queue — jobs section

The jobs section of the Queue page shows every queued / processed file with pagination and per-row cancel/requeue. Background work — library scans, batches, lyrics/cover fetches and scheduled runs — also appears here with a kind badge, a persistent log, and n/total progress. A running task can be stopped from its row (or the stop control on the progress line); work done so far is kept and the job is marked skipped. A thin progress line under the nav (visible on every page) shows the percentage, item counts and the file currently being processed. Bulk controls:

  • Cancel all queued — mark every queued (not-yet-started) job as skipped.
  • Clear completed / errors — delete rows of that status.
  • Clear Review Queue — delete every needs-review row.
  • Clear all — delete every job row that is not currently in-flight.

9. Changes & revert

Every pipeline tag-write records a full pre-write snapshot. From the Changes page you can:

  • Revert — restore the original tags in place and remove the cover.jpg dragontag created. The job can be requeued afterwards to re-tag.
  • Move back — return the file to its pre-pipeline directory. The restored path is automatically added to the excluded files (Settings → Scan filters) so the watcher doesn't immediately re-ingest it.
  • Clear all — wipe the audit log (files are untouched, but revert/move-back is then no longer possible for those entries).

Retention is configurable in Settings → Maintenance (default 500, 0 = unlimited).

10. Schedule

The Schedule tab runs any maintenance task on a standard 5-field cron expression (UTC): scan, organize, the Organize / Re-tag batches, bulk re-tag (with optional dry-run), fetch lyrics, fetch covers, or backup. As you type a cron expression the form shows a plain-English description ("At 06:00, only on Tuesday"), also shown on every schedule row. Each schedule shows its next and last run; Run now fires it immediately, and runs appear on the Queue page like any other task. A run is skipped if the previous run of the same kind is still going.

Examples: 0 3 * * * daily at 03:00 · 0 */6 * * * every 6 hours · 0 4 * * 0 Sundays at 04:00.

11. Backup & restore

Settings → Backup downloads a versioned tarball containing the SQLite database (a consistent snapshot, safe while the app runs), settings.json, the password hash and the AcoustID key, with a manifest and per-file checksums. The last 10 backups are also kept in /config/backups, and backups can be scheduled (§10).

Restoring validates the bundle first (manifest, checksums, database integrity check, settings schema) and refuses while jobs are active; the previous files are kept beside the new ones as *.pre-restore. A container restart afterwards is recommended. If the UI can't boot, restore from the command line:

python -m dragontag.tools.restore_backup /config/backups/dragontag-backup-XXXX.tar.gz

12. Smart formatting

Three independent toggles under Settings → Smart formatting:

  • Title Case — capitalizes the first/last word and any non-article/preposition. Acronyms are preserved.
  • Wrap trailing qualifiers — "Song Live" → "Song (Live)" and similar for Remix, Edit, Demo, Acoustic, etc.
  • Grammar correction — fixes ALL-CAPS strings, inserts missing apostrophes in common contractions and possessives, and normalizes spacing around punctuation. Example: "I DONT LIKE PEOPLES SHIT" → "I Don't Like People's Shit".

File formats

Supported: .flac .mp3 .wav .m4a .mp4

FLAC uses Vorbis Comments. MP3/WAV use ID3v2.4 with TXXX frames for custom fields. MP4/M4A use standard atoms with ----:com.apple.iTunes:<NAME> for custom fields — Picard-compatible.

Template tokens

tokenexample

Review reasons

  • low_score — Best candidate scored below threshold
  • no_match — Neither MB search nor AcoustID returned anything
  • destination_conflict — Target path already exists
  • cover_fetch_failed — Cover Art Archive was unreachable; retry to fetch art
  • dry_run — Dry-run mode is active

Webhook

Configure a Discord webhook (or any HTTP endpoint) in Settings → Webhook. Use the Test button to verify connectivity. Payloads are Discord-compatible embeds sent via HTTP POST and fire on a daemon thread so they cannot block the pipeline.
