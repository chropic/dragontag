Documentation

A reference for everything dragontag does.

Overview

dragontag is a self-hosted audio tagger and library organizer powered by MusicBrainz and AcoustID. Drop audio files into the watched folder (or upload them via the UI) and the pipeline identifies, tags, and moves them into your library automatically.

How it works

Drop file → Read existing tags + filename clues
         → MusicBrainz text search
         → [if empty] AcoustID fingerprint fallback
         → Score candidates → Pick best
         → [score ≥ threshold] Fetch full MB data
         → Fetch cover art (Cover Art Archive)
         → Fetch lyrics (LRCLIB) + detect advisory
         → Write tags to file
         → Move to library destination
         → [score < threshold or missing data] → Review queue

File formats

Supported: .flac .mp3 .wav .m4a .mp4

FLAC uses Vorbis Comments. MP3/WAV use ID3v2.4 with TXXX frames for custom fields. MP4/M4A use standard atoms with ----:com.apple.iTunes:<NAME> for custom fields — compatible with MusicBrainz Picard.

Library organization templates

Configured in Settings → Filename templates. Available tokens:

TokenExample
{track:02d}01, 02, 12
{disc}1, 2
{title}Song Title
{artist}Artist Name
{ext}flac, mp3
{tracktotal}12
{disctotal}2

Review queue

Jobs land in the Review queue when:

  • Low score — Best candidate scored below the auto-apply threshold
  • No match — Neither MusicBrainz text search nor AcoustID returned candidates
  • Missing RELEASETYPE — MB release-group has no primary-type (a heuristic fallback tries first)
  • Destination conflict — Target path already exists
  • Dry run — Dry-run mode is active; no files are written

From the Review page you can pick a different candidate, override the RELEASETYPE, upload custom cover art, or approve/skip the job.

Webhook setup

Configure a Discord webhook (or any HTTP endpoint) in Settings → Webhook notifications. Use the Test button to verify connectivity. Payloads are Discord-compatible embeds sent via HTTP POST.
