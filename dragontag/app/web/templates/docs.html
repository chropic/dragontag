1. Getting started

On first boot you'll be redirected to /setup to create the admin login and (optionally) paste an AcoustID key. After login, head to Library → Manage libraries and add at least one path on disk that dragontag should manage. This becomes the destination root for everything you tag.

Three ways to feed files into the pipeline:

  • Upload — drag files into the Dashboard upload card.
  • Drop folder — copy files into the path mounted at /drop; the watcher catches them.
  • Full library re-tag — point a server-side path at the bulk-retag form to enqueue every audio file inside.

2. Tagging workflow

Drop file  →  Read existing tags + filename clues
           →  MusicBrainz text search
           →  [empty] AcoustID fingerprint fallback
           →  Score candidates  →  Pick best
           →  [score ≥ threshold] Fetch full MB data
                                  Fetch cover art (Cover Art Archive)
                                  Fetch lyrics (LRCLIB) + detect advisory
                                  Apply smart formatting (optional)
                                  Write tags  →  Move to library
           →  [score < threshold or missing data] Review queue

High-confidence matches flow through without input. Anything else lands in Review where you confirm or override.

3. Dashboard

  • Stats bar — total tracks, albums, artists currently indexed.
  • Tag audio files — upload individual files or kick off a folder re-tag.
  • Library stats — top artists, explicit-count, lyrics-coverage, average track length.
  • Recent jobs — last five jobs with status, click any row for full detail.

4. Settings

Hover the ? badge beside any field for an inline tooltip. Highlights:

  • Identification — AcoustID toggle, auto-apply threshold, watcher on/off, global dry-run.
  • Smart formatting — Title Case, qualifier wrapping, grammar correction (see §8).
  • Filename templates — single-disc, multi-disc filename, and multi-disc folder patterns. Use the token chips to insert placeholders; a live preview shows the result.
  • Genre options — cap and casing.
  • Multi-value separators — joiner per tag (default // for ARTIST, ; elsewhere).
  • Skip fields — checkbox grid: any checked field is omitted at write time across every format.
  • Webhooks — Discord-compatible URL with on-done / on-error toggles and a Test button.

5. Library

  • Scan library — refresh the DB from on-disk tags. Run after manually editing tags outside dragontag.
  • Organize library — move existing files to match the current templates. After moves complete, leftover directories that contain no files at all are removed (files are never deleted).
  • Full library re-tag — re-run the full identify pipeline on every file in a folder. Supports a per-run dry-run override.
  • Individual library actions
    • Fetch lyrics only — LRCLIB fetch + embed, no identify.
    • Tag advisories only — re-run the explicit-content classifier on already-embedded lyrics.
    • Fetch cover art only — Cover Art Archive fetch for tracks with an MB album ID.
    • Extract embedded covers — write a cover.jpg from each album's embedded picture when none exists on disk.
    • Recompute ReplayGain — invokes rsgain or loudgain if installed.
    • Verify file integrity — opens every file via mutagen, logs unreadable files.
    • Fix disc folders — normalize Disc N subfolders to the configured template; flatten stray single-disc folders.
    • Find missing tracks — compare local count to the MB release total and report gaps.
  • Track table — click any column header to sort. Use the per-row checkboxes plus "Re-tag selected" to enqueue specific tracks. Choose your page size (10 / 25 / 50 / 100 / 200).

6. Review queue

A file lands in Review when it scored below the threshold, no candidate was found, the destination already exists, RELEASETYPE was missing, or dry-run mode was on. From the Review page you can:

  • Pick any of the top candidates and apply.
  • Override RELEASETYPE, upload custom cover art, or pick a different cover-art thumbnail.
  • Resolve conflicts with Replace, Rename, or Skip.
  • Bulk-select rows and click Apply top candidate to clear them in one go.

7. Jobs

The Jobs page shows every queued / processed file with pagination and per-row cancel/requeue. Bulk controls:

  • Cancel all queued — mark every queued (not-yet-started) job as skipped.
  • Clear completed / errors / needs_review — delete rows of that status.
  • Clear all — delete every job row that is not currently in-flight.

8. Smart formatting

Three independent toggles under Settings → Smart formatting:

  • Title Case — capitalizes the first/last word and any non-article/preposition. Acronyms are preserved.
  • Wrap trailing qualifiers — "Song Live" → "Song (Live)" and similar for Remix, Edit, Demo, Acoustic, etc.
  • Grammar correction — fixes ALL-CAPS strings, inserts missing apostrophes in common contractions and possessives, and normalizes spacing around punctuation. Example: "I DONT LIKE PEOPLES SHIT" → "I Don't Like People's Shit".

File formats

Supported: .flac .mp3 .wav .m4a .mp4

FLAC uses Vorbis Comments. MP3/WAV use ID3v2.4 with TXXX frames for custom fields. MP4/M4A use standard atoms with ----:com.apple.iTunes:<NAME> for custom fields — Picard-compatible.

Template tokens

TokenExample

Review reasons

  • low_score — Best candidate scored below threshold
  • no_match — Neither MB search nor AcoustID returned anything
  • missing_releasetype — Release-group has no primary type (heuristic fallback runs first)
  • destination_conflict — Target path already exists
  • dry_run — Dry-run mode is active

Webhook

Configure a Discord webhook (or any HTTP endpoint) in Settings → Webhook. Use the Test button to verify connectivity. Payloads are Discord-compatible embeds sent via HTTP POST and fire on a daemon thread so they cannot block the pipeline.
